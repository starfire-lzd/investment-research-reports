# 判断条件评分

本文件记录每日复盘中对历史判断条件的三交易日后验证结果。它只保存可复用条件的统计，不记录完整行情流水。

## 评分规则

- `样本数`：条件被明确使用且三交易日后可以验证的次数。
- `成功次数`：条件指向的方向或标的在验证窗口内表现符合原判断，或有效过滤风险的次数。
- `失败次数`：条件指向的方向或标的在验证窗口内表现不符合原判断，或未能过滤风险的次数。
- `成功率`：成功次数 / 样本数。样本数低于 3 时只作观察，不作为强规则。
- `无法验证`：缺少原始计划、价格基准、当前价格或判断对象不明确时，不计入样本数。

## 更新要求

- 每次今日复盘先由模型读取三交易日前的 `盘前分析.md`、`盘中分析.md`、`尾盘分析.md`、`plans/YYYY-MM-DD_交易计划.md` 和 `盘后复盘.md`，抽取需要回看的标的、原始判断摘要和对应条件 ID。
- 模型将抽取结果写入 `data/YYYY-MM-DD/signal-targets.json` 后，再运行 `npm run evaluate:signals`，并读取 `data/YYYY-MM-DD/signal-evaluation.json`。
- 根据当前股价、板块表现、指数表现和计划触发情况，判断当时条件是否准确。
- 准确则按条件 ID 增加对应条件的样本数和成功次数，并写入最近证据。
- 不准确则按条件 ID 增加对应条件的样本数和失败次数，并写入失败模式。
- 无法验证时在今日复盘中说明原因，不更新表格统计。
