# 规则候选池

本文件保存尚未升级为正式规则的研究假设和候选规则。它是交易系统成长的主入口：所有新想法先在这里经历验证，不能因为单日表现直接写入 `rules.md` 或 `risk.md`。

## 生命周期

规则状态只能小步推进：

```text
观察中 -> 候选规则 -> 待验证规则 -> 正式规则 -> 降级/废弃
```

- 观察中：来自盘前假设、盘中异动、错过机会、失败归因或用户输入，还没有足够验证样本。
- 候选规则：已经有明确条件、对象、验证窗口和失败定义，可以进入每日跟踪。
- 待验证规则：已有至少 3 个可验证样本，等待再验证或人工确认是否升级。
- 正式规则：经用户确认后写入 `rules.md` 或 `risk.md`。
- 降级/废弃：被连续证伪、条件不可执行、样本无法验证或与更高优先级规则冲突。

## 升级标准

- 至少 3 个可验证样本。
- 成功率不低于 60%，或失败案例能明确帮助修正规则边界。
- 有清晰适用场景、触发信号、执行动作、风控条件和复盘要求。
- 有明确验证窗口，例如 T+1、T+3、T+5。
- 不与现有已确认风险约束冲突。
- 升级到 `rules.md` 或 `risk.md` 前必须得到用户明确确认。
